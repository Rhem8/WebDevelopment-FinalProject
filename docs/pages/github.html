Git & GitHub

This section covers version control with Git and GitHub — tools that let developers track changes, collaborate on code, and deploy websites to the web.

Git Basics & Version Control

The first part of the lesson explores tracking code changes locally and understanding the lifecycle of file states in Git.

Topics included:

  • What is Version Control: Tracking history, reverting errors, and safe collaboration.
  • The 3-Stage System: Working Directory, Staging Area, and Local Repository.
  • Core Commands: git init, git status, and git log.
  • Saving Progress: Staging with git add and committing with git commit.
  • Remote Repos: Downloading existing repositories with git clone.

Basic Local Git Workflow

# 1. Start a new local repository
git init

# 2. Check which files have changed
git status

# 3. Stage all changes for the next commit
git add .

# 4. Commit the staged changes with a message
git commit -m "feat: initial portfolio layout"

# 5. View the commit history
git log --oneline

This workflow takes a local change from your workspace, stages it, and records it into Git history.

GitHub, Branching & Deployment

The second half covers moving from a local machine to the cloud for team collaboration and live hosting.

Topics covered:

  • GitHub Platform: Repositories, visibility (public vs. private), and forks.
  • Repository Essentials: README.md documentation and .gitignore for secrets.
  • Safe Branching: git branch and git checkout -b for isolated features.
  • Code Integration: Pull Requests and git merge.
  • GitHub Pages: Free hosting for static HTML/CSS/JS websites.

Branching & Deployment Commands

# Create and switch to a new feature branch
git checkout -b feature/contact-form

# [Make your code changes here...]

# Commit changes to the feature branch
git add .
git commit -m "feat: contact form with validation"

# Switch back to main and merge the feature
git checkout main
git merge feature/contact-form

# Push to GitHub to trigger deployment
git push origin main

Working on a separate branch keeps the main codebase safe. Once tested, the branch is merged and pushed live.
